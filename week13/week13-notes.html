How HTML Forms Work

Web forms allow users to provide feedback, orders, and other information. This lesson focuses on creating front-end forms, not server-side handling, and requires programming language knowledge.

An HTML form is a web page where users input information for sending to a database or third-party system. Understanding the server-side process is crucial, and web hosting providers often offer back-end scripts for simple form processing.

A form has a submit button, which sends user information to a URL for processing. Additional technical knowledge is needed for more complex tasks like online store security. Check your web hosting provider's user guide for available form-processing scripts, such as Perl or PHP.

Creating a Form

Every form must begin with an opening <form> tag, which can be located anywhere in the body of the HTML document. The <form> tag typically has three attributes, name method and action

<form name="my_form" method="post" action="myprocessingscript.php">

The most common used method is post, which sends the result of the form entry as a document. Use method="get" to submit results as part of the URL query string in certain cases.

The action attribute allows users to specify the address for sending form data, either to a form-processing program or to their email address, depending on their computer configuration.

Accepting Text Input

Use the <input> tag within a form to ask for specific information, ideally between'and'tags. Place elements alongside text, images, and other HTML tags for easier code comprehension.

<input type="text"id="the_name"name="the_name" placeholder="Enter your full name." size="50"required autofocus>

Input elements must be named and processed by a back-end script, identifying each item as a variable with a value. Form-processing scripts work with variable names and values.

Labeling Each Piece of Form Data

Labeling form data is different from naming or identifying form elements for later usage. Instead, the <label></label> tag pair contains text that serves as a form element caption. A label element in a form element is crucial for screen-reader software since it gives context.

<label class="question" for="the_name">What is your name?</label>

Grouping Form Elements

The <fieldset> element frames form elements to offer context for users, whether using a web browser or screen-reader software. The <fieldset> element establishes the grouping, whereas the <legend> element displays or reads the corresponding text.

Including Hidden Data in Forms

To send data items to a server script without user visibility, use an <input> tag with a type="hidden" attribute. This attribute adds name and value to form results. If using a web hosting provider's form-processing script, specify where to email results.

<input type="hidden"name="mailto"value="me@mysite.com">

Using HTML5 Form Validation

HTML5 has revolutionized web development by introducing form validation, eliminating the need for complex JavaScript-based validation. Forms are automatically validated unless the novalidate attribute is used in the <form> element. Fields are validated for content and type, not just content.

The pattern attribute of the <input> field allows you to specify your pattern-matching requirements using regular expressions, ensuring your input element only contains numbers and letters.

<input type="text" id="the_text" name="the_text" placeholder="Please enter only letters and numbers!" size="50" pattern="[a-z,A-Z,0-9]" required>

Submitting Form Data

Forms often include buttons that transmit form data to a server script or initiate JavaScript actions. Label the Submit button with any label using the value attribute. You may also add a Reset option to let people start again if they change their minds or make errors.

Accessing Form Elements with JavaScript

The form object's elements array, which includes objects for each form element, is its most essential feature. You may refer to an element by name or array index.

The length property of forms and elements can be used to determine the number of objects in an array, while the W3C DOM can be used to access form elements by using an id attribute on the form element.
