Scripting Best Practices

As you develop complex scripts, it's crucial to follow best practices for JavaScript. These guidelines ensure code is readable, easy to maintain, follows standards, works even without JavaScript, adapts easily to different sites, and users appreciate the ease of use and fixability. These best practices are essential for creating AJAX web applications and enhancing pages with simple scripts.

Avoid overusing JavaScript in your website design. As HTML5 and CSS3 mature, there's less need for JavaScript for enhanced interactions. Instead, use CSS for visual effects, familiarity with user interfaces, and compatibility with outdated computers and browsers. Avoid using JavaScript for search engine page rank and mobile-friendly design, as it may penalize your site.

Reading Browser Information

JavaScript's navigator object allows scripts to determine if to use JavaScript elements based on the user's browser, such as implementing code specifically for older browsers. The navigator object, which is not part of the DOM, can be directly accessed and contains various properties that provide information about the browser.

navigator.appCodeName The property specifies the browser's internal code name, like Mozilla, which all modern browsers will return, regardless of whether they are a Mozilla-based browser or not.
navigator.appName Specifies the browser’s name, such as Firefox or Microsoft Edge.
navigator.appVersion Specifies the version of the browser being used—for example, 5.0 (Windows).
navigator.userAgent It specifies the user agent header, which the browser delivers to the web server when requesting a page. Example: Mozilla/5.0 (Macintosh; Intel Mac OS X 10.13; rv:59.0) Gecko/20100101 Firefox/59.0.
navigator.language Specifies browser language (English or Spanish). Codes like en-US for U.S. English are preserved.
navigator.platform Current browser platform. Short strings include Linux i686, Win32, and MacIntel. This allows you to enable platform-specific capabilities like Windows Internet Explorer ActiveX components.

Cross-Browser Scripting

Detecting different browser versions can be simplified by writing cross-browser scripts without referring to the navigator object. This is better as browser-checking code is often confused by new browsers. Feature sensing, a better approach, checks for specific features, like document.getElementById, rather than a specific browser.

The navigator object is useful in situations where a complex script fails in multiple browsers due to a browser bug or feature difference. To fix this, double-check for bugs in your code, use feature sensing to check for missing features in one browser, and use the navigator object to detect a specific browser and substitute code that works there.

Creating an Unobtrusive Script

Unobtrusive scripting involves creating a JavaScript script that enhances a page's functionality without compromising its performance in older browsers, such as creating graphic check boxes as an alternative to regular check boxes.

Using Third-Party JavaScript Libraries

JavaScript's built-in Math and Date functions are often used, but third-party libraries can be used to perform complex tasks with minimal code. However, there are common objections to using third-party libraries, such as not fully understanding the code's workings and the large amount of code that the browser downloads.

It is essential to understand the code behind the scenes and consider the extraneous code if bandwidth limitations or features are disproportionately large. For example, if a library requires a 1MB load for a single function, it may be worth forking or finding a smaller library with less overhead.

jQuery

jQuery, introduced in 2006, is a powerful cross-browser JavaScript library that simplifies DOM manipulation and productivity. It is used in many JavaScript frameworks. The current version is 3.3.1, and jQuery UI offers advanced user interface extensions for rapid development. jQuery Mobile is a touch-optimized mobile framework for responsive apps and websites.

Adding JavaScript Effects by Using a Third-Party Library

Including library functions in the <head> section of a document allows them to be accessible to other scripts or event handlers throughout the page. Ideally, libraries should be placed at the end of the document, but can be placed below all HTML.

To activate the effects, include JavaScript in the <p> tag with the id testarea.

<button onclick="new Effect.Fade($('testarea'))">Fade Out</button>

The code utilizes the $ function in Prototype to obtain the element with ID testarea and then passes it to the Effect.Fade function in script.aculo.us.

Using JavaScript Frameworks

JavaScript libraries are smaller, pre-made code for custom architecture, while frameworks are larger, more complex, and impose an architectural pattern like the model-view-controller (MVC) pattern.

Model - The central component, despite being listed first, holds application data, business rules, functions, and other logical elements.
View - Requests information from the model to show to the user
Controller - Sends information to the model for processing through user interactions

In a web-based application, the user interacts with a controller that manipulates the underlying model, updating the view in the web browser. In traditional web-based applications, both the model and controller components are on the back end, invoked through form elements or interactions. In JavaScript-based MVC applications, all three components can be on the client side, allowing users to interact with data stored and manipulated within the front end, or using AJAX requests.

If you're making a mostly read-only website and just using some JavaScript or jQuery for display, a framework might be too much. If you start thinking about adding user interactivity to your website, consider setting up a framework to help with that. Here are some popular JavaScript frameworks that are great starting points for exploration:

  • Angular (https://angular.io) - This framework is really powerful and flexible, but it has a tough learning curve. It has a really active user community that helps new developers get to know the framework. It's a total revamp of the well-known original framework (https://angularjs.org/)
  • Backbone.js (https://backbonejs.org) - This framework has been around for quite some time (relatively speaking) and served as the inspiration for many other frameworks. It enables a new developer to get started quickly, but the downside of that, for some, is that the applications will contain a lot of unused templating code.
  • Ember (https://emberjs.com) - Ember helps new developers jump in fast, just like Backbone.js. Some might think it’s “too magical,” but Ember’s focus on common programming practices can really help new developers.
  • Knockout (https://knockoutjs.com) - Knockout isn't as popular as the other frameworks mentioned, but it still offers a solid alternative and has some great tutorials for beginners.
  • React (https://react.dev) - People argue about whether React is a library or a framework, but it’s definitely a popular tool for making interactive web pages and apps, and it keeps getting more popular.
