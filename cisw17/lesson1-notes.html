Lesson 1: Understanding How the Web Works

    What You'll Learn in this Lesson

  • A very brief history of the World Wide Web
  • What is meant by the term web page and why this term doesn’t always reflect all the content involved
  • How content gets from your personal computer to someone else’s web browser
  • How to select a web hosting provider
  • How different web browsers and device types can affect your content
  • How to transfer files to your web server by using FTP
  • Where files should be placed on a web server
  • How to distribute web content without a web server
  • How to use other publishing methods, such as blogs
  • Tips for testing the appearance and functionality of web content

Sir Tim Berners-Lee developed the Hypertext Markup Language (HTML) to cross-reference text over the internet. By 1993, 100 computers were capable of processing HTML pages, and this network of computers was dubbed the world wide web. Web browsers, a specific type of appllications, are developed to enable people to view web pages. As popularity of the web grew, web browsers also grew to allow more types of media to be displayed. The delivery of web based content to users involves several processes that occurs in different locations, and these processes could occur multiple times between the browser and the server before the full webpage could be displayed. While web pages could be viewed locally, having a web hosting provider is recommended. Finding and utilizing a web hosting provider is usually an easy task, and some criterias like reliability/server "uptime," customer service, server space, bandwidth, domain name purchase and management, and price, should be considered when looking for a good provider. In order to ensure all visitors could properly view a webpage, it is essential to test with multiple different web browsers and devices. This is because different browsers could display different content differently, which could result in a different appearance than what was expected. The process of transferring files to a web server uses a client called the FTP Client, which uses the File Transfer Protocol to transfer files from your local computer to a web server. Maintaining web content involves organizing files into directories, making management of the webpage more efficient.

HTML Tags Every Webpage Must Have

The first line of code should be <!doctype html>, which indentifies the document as using HTMl5. An opening tag (< >)indicates the start of a HTML command, while a closing tag (</ >) indicates the end of a command. An empty tag issues commands without any enclosing text within the tag. The body tag tells the browser where the actual body content is. The top of the browser window display the text that is located in the title tag. The character set used by the webpage is declared by the tag <meta charset="utf-8">. Body and titles tags should be used in every page as it is required on every webpage.

Tag Description
<html>…</html> Encloses the entire HTML document
<head>…</head> Encloses the head of the HTML document, located within the HTML tag pair
<meta charset="utf-8"> Defines the character set for the document, which in this case is UTF-8
<title>…</title> Indicates the title of the document/webpage. Located within the head tag pairs
<body>…</body> Encloses the body of the HTML document, within the html tag pairs
<p>…</p> Encloses a paragraph & skips 1 line between paragraphs
<br> Displays a line break
<hr> Displays a horizontal rule
<h1></h1> Encloses a 1st level heading
<h2></h2> Encloses a 2nd level heading
<h3></h3> Encloses a 3rd level heading
<h4></h4> Encloses a 4th level heading
<h5></h5> Encloses a 5th level heading
<h6>…</h6> Encloses a 6th level heading
<header>…</header> Contains introductory information
<footer>…</footer> Contains supplementary information for the webpage, like copyright or author information
<nav>…</nav> Contains navigational elements for the webpage
<section>…</section> Contains contents similar in theme
<article>…</article> Contains content that is a standalone body of work
<aside>…</aside> Contains secondary element for its parent element
<address>…</address> Contains address information for its nearest article or body element
