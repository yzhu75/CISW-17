Lesson 16: Understanding the Importance of Responsive Web Design

What You'll Learn in this Lesson

  • What is responsive web design (RWD)
  • Why we need RWD
  • What is progressive enhancement
  • How to write HTML that’s RWD ready

Responsive web design (RWD) optimizes viewing on the customer's device. RWD helps designers and site owners create “edit once, display anywhere” websites. The web designer creates one page that works on a wide-screen desktop or a small-screen cell phone without switching sites. Responsive websites adjust their layout to fit different devices. RWD makes these changes without scripts or programming, unlike older solutions. Instead, RWD defines fluid grids, variable font sizes. Progressive enhancement involves building a strong website foundation and adding enhancements as browsers and devices can handle them. Responsive design is easy to learn because it adds no HTML tags or attributes. As mentioned, you write well-formed, valid, and semantic HTML. HTML5, the latest version of HTML, helps web designers use progressive enhancement and RWD the most. You can build RWD sites with older HTML, but it's best to stay current.

Validate HTML, CSS, and JavaScript as you build a progressive enhancement site for responsive web design. HTML validation is simple and quick. Enter the URL to be validated at https://validator.w3.org. Upload a file or paste the HTML directly into the page to validate it. Jigsaw (https://jigsaw.w3.org/css-validator/) validates CSS similarly.


Lesson 17: Designing for Mobile Devices

What You'll Learn in this Lesson

  • How mobile design differs from standard web design
  • What the Mobile First design philosophy is
  • How Mobile First is important to RWD
  • How to build responsive tables and images for mobile devices
  • How to use CSS columns to create responsive designs without media queries

Around October 2016, Statcounter reported that mobile usage surpassed desktop usage worldwide. While desktops still outnumber mobiles in North America, almost 40% of the market is mobile. Tablets plus mobile bring that number closer to 50%. If you ignore or downplay mobile, you're losing nearly half of North American and a majority of global audiences. Designing a mobile site requires more than testing on your phone after launch. Best mobile designs treat mobile devices as important as desktop computers, if not more so. Traditional web designers built a desktop site first, then modified it for mobile. Mobile First design reverses that formula: build a site for mobile users first, then for desktop users. Mobile First starts with creating the default mobile design for all your pages. Content comes first when designing for mobile devices.

Responsive websites face challenges with large tables and images, which can be difficult to handle on small devices and cause slow page loading. To make a table responsive, consider essential data, customer comparison, and screen sizes, as well as the necessary dimensions for legibility. Mobile First design ensures consistent data on all devices, hiding less important information until users need it. Responsive design should reflect data comparisons or make information digestible. CSS columns can be used to create responsive layouts, a technique often overlooked in creating layouts on different devices. Responsive web design (RWD) is not the only solution for mobile design. Alternatives include adaptive design, dynamic serving, separate sites for mobile, and creating apps. RWD can be challenging due to its one-size-fits-all approach, which may slow down pages. Content management is another challenge, as mobile customers have different goals and desires. RWD relies on modern, up-to-date browsers and devices, which may cause slower or non-functional pages on older devices.


Lesson 18: Using Media Queries and Breakpoints

What You'll Learn in this Lesson

  • How to write a CSS media query
  • How to use different media types
  • How to create media query expressions
  • Understanding logical keywords in your queries
  • What CSS breakpoints are
  • How to define breakpoints
  • How to find the best breakpoints for your website

The W3C defines a media query as a logical expression that is true or false based on the device's media type, allowing web designers to define if/then statements. CSS media types and media queries were introduced in CSS2, with media queries becoming a full-fledged recommendation in 2012. Modern browsers and user agents support these stable tools. To effectively use media queries, one must master writing expressions, including basic expressions targeting browsers above, below, and between minimum and maximum widths, as well as more complex ones. Media query expressions utilize logical operators to define intricate scenarios, including the following operators:

  • and - The query matches all elements, such as a device in landscape mode with a 768px browser window, by combining features and types.
  • Comma-separated list - The OR logical operator matches features or types in a comma-separated list, matching both landscape and portrait browsers if any are true.
  • not - The query will match if the query would normally return false.
  • only - This prevents browsers that don't support media queries from applying the styles.

CSS breakpoints are device features with media query declarations, adjusting design width to accommodate different devices. Most responsive designs have at least one breakpoint affecting design appearance. When choosing breakpoints for a website, it's crucial to consider that each additional breakpoint increases the cost of building and maintaining the website. CSS media queries define breakpoints based on device width, with three breakpoints for a Mobile First site: one for every browser, one for 480px to 1200px widths. Breakpoint numbers for RWD depend on customer and design. View site at different widths and use Responsive Design Mode to identify design breaks. Best practices for choosing breakpoints include choosing for design rather than device resolution, keeping breakpoints 1px different, considering device orientation, and optimizing for Retina devices.
